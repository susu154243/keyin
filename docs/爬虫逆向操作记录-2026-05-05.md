## 一、前期逆向分析（5月4日-5日凌晨）

### 1.1 目标站点概况
- **网址**：https://ruankaodaren.com/exam/#/
- **架构**：Vue.js SPA（单页应用）
- **API 基础路径**：`/api`
- **租户配置**：`tenantId: "1"`

### 1.2 JS 逆向发现
- 下载并去混淆主 `app.js` 包
- 提取约 1195 个解码字符串
- 定位全部 API 端点

### 1.3 端点分类

**公开接口（无需 token）：**
| 端点 | 说明 |
|------|------|
| `GET /api/knowledge/list` | 知识分类列表 |
| `POST /api/user_info/phone/login` | 手机验证码登录 |

**需认证接口（无 token 返回 401 invalid token）：**
| 端点 | 说明 |
|------|------|
| `POST /api/chapter/list` | 章节列表 |
| `POST /api/question/chapter/practice/list` | 章节练习题列表 |
| `POST /api/question/practice` | 练习题详情 |
| `POST /api/login` | 登录端点 |
| `POST /api/user/login` | 用户登录端点 |

### 1.4 认证机制
- **认证方式**：JWT Token（Bearer Auth）
- **Token 获取**：从已登录浏览器提取 JWT token，绕过微信登录障碍
- **Token 刷新**：强制刷新 20003 错误需重新提取

## 四、题型展示方案设计

### 4.1 三类题型处理策略

| 题型 | 展示模式 | 说明 |
|------|----------|------|
| 单选题/多选题 | 标准选择题 | 沿用现有逻辑 |
| 案例分析题 | 学习卡片 | category_id=26 识别，自评走 SM-2 |
| 论文题 | 学习卡片 | category_id=27 识别，自评走 SM-2 |

### 4.2 学习卡片模式设计
- 不计入评分
- 不纳入错题和强化队列
- 不影响 FSRS 调度
- 前端展示：题干全宽 + 查看答案 + 参考答案 + 自评按钮（已掌握/没记住）
- 通过 `question.options` 是否为空区分常规选择题和学习卡片

## 六、踩坑记录

| 问题 | 原因 | 解决 |
|------|------|------|
| 微信登录受阻 | 自动化爬取被微信拦截 | 从浏览器提取 JWT token 绕过 |
| 案例分析子题重复 | 每个子题携带完整【说明】文本 | 合并同一案例的子题（134→36） |
| 论文题无标准答案 | 主观论述题不适合选择题格式 | 设计学习卡片模式 |
| 历年真题 API 500 | 服务端返回 500 错误 | 仅获取元数据，暂不处理 |
